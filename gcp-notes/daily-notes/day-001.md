## Q-1. What is an operating system?

- OS = kind of a boss that controls everything inside of computer. Ex : LINUX.
- OS is a kind of software that acts as an inteface between  the computer hardware and the user.
- Manages hardware resources and provides services for application program. 

### KEY FUNCTIONS :
1. Process management 
2. Memory management
3. File sysytem 
4. Device management
5. Security access and control

### IMPORTANT LINES :
- 'without an os user can't interact with hardware directly and applicatios can't run efficiently.'
- 'os abstracts the hardware complexity and provides standardized interface through system calls.'
- 'system calls acts as an controlled gateway between user applications and hardware via os'

----

## Q-2 What is a linus kernel?

- Linux kernel is the main boss that controls hardware and lets program use it safely.
- Core component of linux os that directly interacts with hardware.
- Manages system resources such as CPU, memory and devices.
- Provides interface for user applications through system calls.

----

## Q-3 What is a shell? 

- Shell is translator between you and kernel
- Converts user commands into system calls that kernel understands
- CLI (command line interface)- Terminal, most common in linux
- GUI (graphical user interface)- Buttons, windows

----

## Q-4 User space vs Kernel space

- User space is where user applications runs with limited access.
- Kernel space is where OS kernel executes with full access to hardware.
- Separation done to ensure safety and stability of a system.

| Feature | User space | Kernel space |
| Access  | Limited    | FUll         |
|Runs     | Applications| OS Kenel    |
| Safety  | Safe        | Critical    |

## Q-5 How does a linux command actually work?

You type - shell understands - kernel executes - output comes back
- Shell uses system calss like fork() to create new processes
- exec() to execute commands
- Kernel schedue=les and executes the processes
- fork() creates new processes exec() allows that program to run without disrupting the parent process

## Q-6 Terminal vs Shell

- TERMINAL = UI that allows the users to interact with system by writing commands. Provides a way to  access shell.
- SHELL = Program that interprets the commands, turn them into system calls and sends the kernel for execution.

## Q-7 What is BASH?

- BASH (Bourne Again Shell)
- command line shell and scripting language in linux systems
- interprets user commands, executes program and support scripting for automation
- default shell in many linux distributions

### FLOW:

user - terminal - bash - program - system call - kernel

### Shell comparison:

| Shell | Strength | Weakness |
|BASH   | universal,stable | less fancy|
|zsh | customizable, smart | slightly heavy |
|fish | user-friendly | NOT-POSIX |
|DASH | fast | limited features |

### POSIX:
- portable operating system interface.
- Set of standards for how unix-like system and shells should behave.
- POSIX compliant shells ensure portability of scripts across diffrent systems.
